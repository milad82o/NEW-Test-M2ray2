Setup Guides

Step-by-step instructions for deploying your g2ray proxy.

Quick Start

Deploy your g2ray proxy using GitHub Codespaces.

  1. 1 Fork github.com/amircloner/g2ray
  2. 2 Click Code → Codespaces → Create codespace on main
  3. 3 Wait 2-5 minutes for Xray to install
  4. 4 Copy the VLESS link from terminal or use show-link.sh

Files in .devcontainer/

config.json — Xray config: VLESS + xHTTP + TLS + sniffing + routing + DNS

setup.sh — Downloads Xray + GeoIP + GeoSite

Dockerfile — Container build (base: debian:bookworm-slim)

devcontainer.json — Codespace settings: port 443 public, CLI features

show-link.sh — Prints VLESS link (reads UUID from config)

start.sh — Runs Xray in tmux session

SNI Format

GitHub port-forwarder issues TLS certs for:

<codespace-name>-<port>.app.github.dev

Show Link

If the link isn't shown in the terminal:

show-link.sh

Or extract manually:

UUID=$(grep -o '"id": *"[^"]*"' /etc/xray/g2ray.json | head -1 | grep -o '"[^"]*"$' | tr -d '"') echo "vless://${UUID}@94.130.50.12:443?..."

Troubleshooting

Connection times out → try ws/h2/grpc transport + fragment

Slow speeds → use xhttp-go mode

TLS error → verify SNI matches codespace name

Quota used → stop codespace when not in use

Idle timeout → set github.com/settings/codespaces → 240 min

Start Xray manually → bash /usr/local/bin/start.sh
